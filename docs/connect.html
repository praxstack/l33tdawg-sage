Connect Your AI to SAGE

SAGE is installed. Now give your AI access to its memory. Pick your provider below.

Before you start

  1. SAGE is installed — you've run the app or sage-lite setup
  2. SAGE is running — double-click the app, or run sage-lite serve in a terminal
  3. You should see the Brain Dashboard at http://localhost:8080/ui/
1

Find your sage-lite binary path

If you installed from the DMG, it's inside the app bundle. If you built from source, it's wherever you put it.

# DMG install (macOS):
/Applications/SAGE.app/Contents/MacOS/sage-lite

# Built from source:
which sage-lite
2

Add MCP config to your project

Create a .mcp.json file in your project root (or home directory for global config):

{
  "mcpServers": {
    "sage": {
      "command": "/Applications/SAGE.app/Contents/MacOS/sage-lite",
      "args": ["mcp"],
      "env": {
        "SAGE_HOME": "~/.sage",
        "SAGE_PROVIDER": "claude-code"
      }
    }
  }
}
SAGE_PROVIDER scopes your memories. Claude Code memories stay separate from ChatGPT memories. Facts (verified truths) are shared across providers.
3

Restart Claude Code

Close and reopen Claude Code (or start a new session). You should see SAGE tools available: sage_turn, sage_recall, sage_remember, etc.

4

Initialize your AI's brain

In your first conversation, tell Claude:

Call sage_inception to initialize your memory.

That's it. Claude will now remember things across conversations.

1

Open Claude Desktop settings

Go to Claude Desktop → Settings → Developer → Edit Config (under MCP Servers).

This opens the file claude_desktop_config.json.

2

Add the SAGE MCP server

Paste this into the config file:

{
  "mcpServers": {
    "sage": {
      "command": "/Applications/SAGE.app/Contents/MacOS/sage-lite",
      "args": ["mcp"],
      "env": {
        "SAGE_HOME": "~/.sage",
        "SAGE_PROVIDER": "claude-desktop"
      }
    }
  }
}
Windows? Change the command to:
C:\\Program Files\\SAGE\\sage-lite.exe
3

Restart Claude Desktop

Completely quit and reopen Claude Desktop. Look for the SAGE tools icon in the chat input area — you should see tools like sage_turn and sage_recall.

4

Initialize

Start a new conversation and say:

Call sage_inception to initialize your memory.

Claude will seed its foundational memories and start learning from every conversation.

Option A: Native MCP (Paid plans)

Requires ChatGPT Plus, Pro, Team, or Enterprise. Native MCP support via Developer Mode is not available on the free ChatGPT plan.
1

Enable Developer Mode

Go to Settings → Beta Features → enable Developer Mode. Then in Settings → Developer, add an MCP server.

2

Add SAGE as an MCP server

In ChatGPT's MCP configuration, add:

{
  "mcpServers": {
    "sage": {
      "command": "/Applications/SAGE.app/Contents/MacOS/sage-lite",
      "args": ["mcp"],
      "env": {
        "SAGE_HOME": "~/.sage",
        "SAGE_PROVIDER": "chatgpt"
      }
    }
  }
}
Windows? Change the command to:
C:\\Program Files\\SAGE\\sage-lite.exe
3

Restart ChatGPT and initialize

Quit and reopen the ChatGPT desktop app. SAGE tools should appear in the tools menu. Start a new conversation and say:

Call sage_inception to initialize your memory. Then call sage_turn every time I send you a message so you build persistent memory across our conversations.
Tip: ChatGPT doesn't have a CLAUDE.md file to remind it. Add "Always call sage_turn at the start of each reply" to your Custom Instructions (Settings → Personalization) so it remembers automatically.

Option B: Chrome Extension (Free & Paid plans)

Works on all ChatGPT plans including free — no desktop app required. Uses a Chrome extension to bridge MCP tools to the ChatGPT website (chat.openai.com). Note: the extension can see your chat content.
1

Install MCP SuperAssistant

Install the MCP SuperAssistant Chrome extension. It detects MCP tool calls from AI chats and executes them locally.

2

Configure SAGE as an MCP server

Open the extension's sidebar and add a new MCP server with the same config:

Command: /Applications/SAGE.app/Contents/MacOS/sage-lite
Arguments: mcp
Environment:
  SAGE_HOME=~/.sage
  SAGE_PROVIDER=chatgpt
3

Use ChatGPT as normal

Open chat.openai.com in Chrome and chat normally — no desktop app needed. The extension intercepts MCP tool calls, runs them against your local SAGE instance, and injects the results back into the chat. Tell ChatGPT:

Call sage_inception to initialize your memory. You have SAGE MCP tools available — use sage_turn every reply to build persistent memory.

Troubleshooting

"SAGE tools don't appear"

Make sure sage-lite serve is running (or the SAGE app is open). Check that the path to sage-lite in your config is correct. Try running the command manually in a terminal to verify it works.

"Connection refused" or "embedding provider error"

SAGE needs to be running on port 8080. Check http://localhost:8080/ui/ in your browser. If the dashboard loads, SAGE is running. If not, start it with sage-lite serve or open the SAGE app.

"sage_inception fails"

This usually means the embedding provider isn't configured. Run sage-lite setup to re-run the setup wizard and pick an embedding provider (Ollama recommended, or Hash for zero-setup).

Can I use SAGE with multiple AI providers at the same time?

Yes! That's exactly what SAGE_PROVIDER is for. Each provider gets its own memory scope. Set "SAGE_PROVIDER": "claude-code" in one config and "SAGE_PROVIDER": "chatgpt" in another. Facts are shared across providers, but observations and reflections stay scoped.

Where is my data stored?

Everything stays on your machine in ~/.sage/. The database is a SQLite file. Nothing is sent to any cloud service. You can export your entire memory vault with sage-lite vault export.
