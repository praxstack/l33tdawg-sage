Privacy First

Your data never
leaves your machine

Most "memory plugins" send your conversations to servers you've never heard of. SAGE runs entirely local. A SQLite file in your home directory. That's it.

🔒
🏠

100% Local

No cloud accounts. No telemetry. No API keys phoning home. Your memories live in ~/.sage/ and nowhere else.

👁

Fully Inspectable

Standard SQLite database. Open it with any tool, query it, back it up, move it between machines. You own every byte.

🔐

Tamper-Proof

Every memory is digitally signed with your local key. Full audit trail — you can verify nothing has been changed or injected.

🔒

AES-256 Encrypted at Rest

Optional end-to-end encryption for your memory vault. If your laptop is stolen, backed up to iCloud, or synced to Google Drive — nobody can read your memories without your passphrase.

Switching Providers?

Take your memories
with you

Changing AI providers shouldn't mean starting from zero. SAGE imports your conversation history so your new AI already knows your projects, your preferences, and your way of working.

💬

Coming from ChatGPT

Export your conversations from OpenAI, then let SAGE turn them into structured memories your new AI can search and learn from.

  1. 1 Go to Settings → Data Controls → Export Data in ChatGPT
  2. 2 Download the ZIP file from the email OpenAI sends you
  3. 3 Run ./sage-lite setup and drag-drop the ZIP into the import step
  4. 4 SAGE parses every conversation into searchable, governed memories
🚀

Coming from any provider

Have conversation logs, notes, or project documentation? SAGE can ingest them too — any text your AI should remember.

  1. 1 Export your data from your current provider (most offer this)
  2. 2 Drop JSON or text files into the SAGE setup wizard
  3. 3 SAGE structures and indexes everything locally
  4. 4 Your new AI starts with full context from day one
Your data stays yours. The import runs entirely on your machine. Nothing is uploaded anywhere. Your old conversations become private, local memories that only your AI can access.
For Developers

Infrastructure,
not a library

SAGE isn't an API you call or a package you import. It's memory infrastructure for AI agents — a local server your AI connects to via MCP. Build from source or use the Python SDK to integrate programmatically.

terminal
# Clone and build
$ git clone https://github.com/l33tdawg/sage.git && cd sage
$ go build -o sage-lite ./cmd/sage-lite/
 
# Setup wizard (opens browser, generates MCP config)
$ ./sage-lite setup
 
# Start SAGE
$ ./sage-lite serve
✨ SAGE Personal ready
  Dashboard: http://localhost:8080/ui/
 
# Or use the Python SDK
$ pip install sage-sdk
